## 1. Executive Summary

The web PWA is ~90% shippable today; the native iOS app is ~15% shippable and gated on weeks of human-only external dependencies.

Two separate products live in this repo:
- **`docs/` — the web PWA.** Real, functional, close to launchable. Complete daily loop (onboarding, habits, timers, wolf growth, stats, journey, profile), focus-trapped modals, service worker, privacy/terms/waitlist, 5-form wolf engine. Outstanding issues are polish, resilience edge-cases, accessibility, and a few real correctness bugs, all agent-fixable. No paywall/ads yet, which is the deliberate Phase-1 plan.
- **`Lupo/` — the SwiftUI native app.** A scaffold. There is NO Xcode project (no .xcodeproj/.pbxproj/Package.swift/Info.plist/.entitlements/Assets.xcassets/PrivacyInfo.xcprivacy). Nothing compiles or archives. The Screen Time differentiator (Family Controls) is commented out and needs an Apple entitlement (1-4+ wk approval). The paywall is a non-functional stub.

**Bottom line:** the native app cannot be submitted in its current state. The true critical-path blockers are all human-required with long lead times: a Mac + paid Apple Developer membership, the Family Controls distribution entitlement (1-4 wk Apple approval), and a committed buildable Xcode project. The fastest real launch is the PWA: polish it, ship it as the waitlist/on-ramp, run the native track in parallel behind the entitlement clock.
